Object Access level

The object your are configuring in Second Life can do a very simple security to check, so that only certain avatars can access it:

  • Public: any avatar is allowed
  • Group (SL): only avatars in the same Second Life group as the object are allowed
  • Owner: only the owner of the object is allowed

There are two types of access you can configure (although not all objects support both of these types):

  • Use object: ordinary access to use the object (e.g. submit objects to the PrimDrop, or participate in a WebIntercom chat)
  • Control object: perform maintenance/administration tasks on the object (e.g. activating or deactivating the WebIntercom)
